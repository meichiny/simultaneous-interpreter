# 显示窗口方式分析

## 当前三种方式

### 方式1：配置页 → 先开显示窗口 → 再运行
**路径**：主配置页 → 点击顶部「📺 显示窗口」→ 打开独立窗口 → 回到主页面 → 点击「开始运行」

**特点**：
- 独立窗口提前准备就绪
- 翻译开始后文字自动同步到新窗口
- 需要用户在两个窗口间切换操作

**问题**：
- 操作步骤繁琐（3步）
- 用户可能忘记先开窗口，已经开始翻译了才发现

---

### 方式2：配置页 → 直接运行（原项目默认）
**路径**：主配置页 → 点击「开始运行」→ 页面内嵌显示翻译界面

**特点**：
- 单窗口操作，简单直接
- 翻译界面替换配置界面，沉浸感强
- 适合专注使用，不适合边配置边观看

**问题**：
- 无法同时看到配置和翻译
- 关闭翻译需要退出到配置页
- 样式固定，无法自定义

---

### 方式3：配置页 → 运行后 → 再开显示窗口
**路径**：主配置页 → 点击「开始运行」→ 切换到翻译界面 → 点击顶部「📺 打开显示窗口」→ 新窗口打开

**特点**：
- 先开始翻译，随时可打开独立窗口
- 两个窗口同时显示相同内容

**问题**：
- 内嵌界面和独立窗口内容重复
- 用户困惑：看哪个窗口？
- 同样存在样式不一致问题

---

## 新想法：配置页预设样式 → 运行后应用

**路径**：主配置页预设样式（字号/颜色/背景）→ 点击「开始运行」→ 翻译界面自动应用这些样式

**优点**：
1. **样式前置**：配置时就看到效果，符合直觉
2. **一步操作**：无需额外点击，运行即生效
3. **统一体验**：方式2的内嵌界面也有自定义样式
4. **降低认知负担**：不需要理解"主窗口/显示窗口"的区别

**缺点**：
- 独立窗口（方式1/3）的样式还需要单独设置
- 如果用户想要两个窗口不同样式，需要额外配置

---

## 分析与建议

### 推荐方案：**新想法 + 简化窗口逻辑**

1. **样式配置前置**：在配置页（第3卡片）增加显示样式设置
2. **样式自动继承**：无论哪种方式，都自动使用配置页的样式
3. **简化操作路径**：
   - 只保留「内嵌模式」和「独立窗口模式」两种
   - 删除方式3的重复逻辑

### 推荐实施步骤

**Phase 1**：新想法（配置页预设样式 → 内嵌界面应用）
- 在主配置页添加样式设置面板
- 修改方式2的内嵌界面，支持动态样式

**Phase 2**：独立窗口优化  
- 独立窗口自动继承配置页样式
- 提供「在新窗口运行」一键按钮

# Phase 2 设计方案

## 1. 运行模式选择（配置页）

在「本地音频」卡片内添加：

```
运行模式 / Run Mode
○ 内嵌运行 Run in Page    ○ 新窗口运行 Run in Window
```

- 默认选择「内嵌运行」
- 保存到 sessionStorage

## 2. "运行中"简化面板设计

当选择「新窗口运行」并点击开始后：

### 方案 A：保持配置页（推荐）
主页面保持配置页不变，只是：
- 「开始运行」按钮变成「停止运行」（红色）
- 按钮旁边显示状态指示器（绿色圆点 + "运行中"文字）
- 底部状态栏显示翻译统计（可选）

```
┌────────────────────────────────────────────┐
│  语言方向    本地音频    术语库              │
│  ┌───────┐  ┌───────┐  ┌───────┐          │
│  │ ...   │  │ ...   │  │ ...   │          │
│  │       │  │ 运行模式: [新窗口]            │
│  │       │  │ ○内嵌 ○新窗口                │
│  │       │  │       │  │       │          │
│  └───────┘  └───────┘  └───────┘          │
│                                             │
│                    [🟢 运行中] [停止运行]   │
└────────────────────────────────────────────┘
         ↓ 独立窗口已打开显示翻译
```

### 方案 B：简化状态面板
主页面切换到一个极简面板：

```
┌────────────────────────────────────────────┐
│  AI Meeting Mate              [🟢 运行中]   │
├────────────────────────────────────────────┤
│                                             │
│     翻译正在独立窗口运行                    │
│     Translation running in display window   │
│                                             │
│     [停止运行]  [返回配置]                  │
│                                             │
└────────────────────────────────────────────┘
```

### 推荐：方案 A
理由：
- 用户可以随时调整配置（如术语库）
- 可以看到当前配置状态
- 可以随时停止运行
- 更符合「配置中心」的设计思路

## 3. 独立窗口样式统一

- 独立窗口 `/display` 只读 URL 参数，不再读 localStorage
- 设置按钮（⚙️）保留，但改为「临时预览」模式
- 关闭独立窗口后，样式不保存（因为配置页是权威来源）
